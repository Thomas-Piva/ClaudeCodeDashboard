/analizzacodebase — come funziona

1. Scansione file sorgente

find . -type f \
  \( -iname "*.vb" -o -iname "*.cs" \
     -o -iname "*.js" -o -iname "*.ts" \) \
  ! -path "*/node_modules/*"

2. Output per ogni file

---
layer: service
depends_on:
- database
- clienti
last_analyzed: 2026-04-24
---

Layer architetturale

api · service · data · ui · utility

Dipendenze dichiarate

Lista dei moduli da cui dipende — usata da /aggiornacodebase per propagare gli aggiornamenti

Regola accuratezza

Se il comportamento non è chiaro dal codice → chiede prima di scrivere. Non inventa mai.

Quando NON usarlo

Modifiche puntuali → usa /aggiornacodebase
È un full-rescan: rigenera tutto

Fase 1 — Classifica file

Nuovo: nessun doc wiki → analizza
